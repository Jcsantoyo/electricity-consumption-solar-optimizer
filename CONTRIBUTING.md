# Contributing

This document describes the recommended development workflow for this project.

The project is mainly a personal learning and portfolio project, but these guidelines help keep the repository reproducible, tested and easy to maintain.

---

## Development setup

Start from a clean clone:

```bash
git clone https://github.com/Jcsantoyo/electricity-consumption-solar-optimizer.git
cd electricity-consumption-solar-optimizer
```

Create and activate a virtual environment:

```bash
python -m venv venv
source venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

or:

```bash
make install
```

For a full setup guide, see:

```text
SETUP.md
```

---

## Running tests

Run the test suite with:

```bash
make test
```

or:

```bash
python -m pytest
```

Tests are located in:

```text
tests/
```

Pytest configuration is defined in:

```text
pyproject.toml
```

---

## Running lint checks

Run Ruff checks with:

```bash
make lint
```

or:

```bash
ruff check .
```

The current Ruff configuration focuses on basic Python correctness checks.

---

## Running the full pipeline

To regenerate the main project outputs, run:

```bash
make pipeline
```

or:

```bash
python scripts/run_full_pipeline.py
```

The full pipeline generates:

- Configuration summary
- Historical optimization results
- Forecasting results
- Forecast-based optimization results
- Historical vs forecast-based comparison
- Final results summary
- Plots

Main outputs are saved in:

```text
reports/
images/
```

---

## Before committing

Before creating a commit, run:

```bash
make lint
make test
```

For changes that affect reports, plots, configuration or pipeline logic, also run:

```bash
make pipeline
```

Then check:

```bash
git status
```

Make sure the files included in the commit match the intended change.

---

## Commit style

Use short commit messages in English.

Examples:

```text
Add final results summary report
Document setup guide
Add Makefile smoke tests
Improve gitignore for development caches
```

Recommended style:

```text
Verb + short description
```

Examples:

```text
Add ...
Update ...
Document ...
Fix ...
Clean ...
Improve ...
```

---

## Generated files

Some files are generated by the pipeline.

Examples:

```text
reports/final_results_summary.md
reports/configuration_summary.md
reports/historical_vs_forecast_optimization.csv
images/historical_vs_forecast_payback.png
```

If a code change affects generated outputs, regenerate them before committing:

```bash
make pipeline
```

Large raw data files are ignored by Git and must be added locally.

---

## Data files

The project expects external data files such as:

```text
data/raw/household_power_consumption.txt
data/raw/pvgis_hourly_linares_1kw_2020.csv
```

These files are ignored by Git because they may be large or external.

See:

```text
data/README.md
SETUP.md
```

for data setup instructions.

---

## Continuous integration

GitHub Actions runs automated checks on push and pull request.

The workflow runs:

```bash
ruff check .
python -m pytest
```

A passing workflow means that the repository installs correctly, passes lint checks and passes the test suite in a clean environment.
