# Reports documentation

This folder contains generated reports produced by the project pipeline.

Most files in this folder are outputs. They can be regenerated by running:

```bash
make pipeline
```

or:

```bash
python scripts/run_full_pipeline.py
```

---

## Recommended files to read first

### `final_results_summary.md`

Readable final summary of the main results produced by the full pipeline.

This is the best file to open first if you want to understand the final conclusions of the project.

It compares:

- Best historical economic scenario
- Best historical self-sufficiency scenario
- Best forecast-based economic scenario
- Best forecast-based self-sufficiency scenario

### `configuration_summary.md`

Summary of the configuration used to generate the results.

It includes:

- Input data paths
- PVGIS settings
- Grid search parameters
- Battery model assumptions
- Economic assumptions
- Active tariff profile
- Tariff values

### `outputs_index.md`

Index describing the generated reports and plots.

This file is useful as a map of the project outputs.

---

## Historical optimization reports

### `grid_search_results.csv`

Full historical optimization grid search results.

Each row represents one tested combination of:

- Solar peak power
- Battery capacity

For each scenario, it includes metrics such as:

- Investment cost
- Annual savings
- Payback period
- Grid import
- Annual grid import
- Solar surplus
- Self-sufficiency

### `best_scenarios.csv`

Summary table with the two selected historical optimal scenarios:

- Best economic scenario by minimum payback period
- Best energy scenario by maximum self-sufficiency

### `best_scenario_timeseries.csv`

Hourly energy flow table for the selected best economic historical scenario.

It includes:

- Consumption
- Solar generation
- Self-consumption
- Battery charge
- Battery discharge
- Grid import
- Solar surplus
- Battery state of charge

### `summary.txt`

Readable text summary of the main historical optimization results.

---

## Forecasting reports

### `forecast_results.csv`

Forecasting results for the test period.

It includes:

- Actual electricity consumption
- Predicted electricity consumption

### `forecast_feature_importance.csv`

Feature importance values from the Random Forest forecasting model.

This helps explain which variables are most relevant for predicting household electricity consumption.

### `forecast_model_comparison.csv`

Comparison between forecasting models using error metrics such as:

- MAE
- RMSE

---

## Forecast-based optimization reports

### `forecasted_consumption_for_optimization.csv`

Forecasted consumption converted into the standard project input format.

This file connects the Machine Learning forecasting module with the solar and battery optimization pipeline.

### `forecast_optimization_results.csv`

Full grid search results using forecasted consumption instead of historical consumption.

### `forecast_optimization_best_scenarios.csv`

Best economic and best self-sufficiency scenarios obtained from forecast-based optimization.

---

## Historical vs forecast-based comparison

### `historical_vs_forecast_optimization.csv`

Comparison table between the best historical optimization scenarios and the best forecast-based optimization scenarios.

It allows checking whether the recommended solar and battery configuration changes when the optimization uses predicted future consumption instead of historical consumption.
